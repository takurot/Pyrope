# Benchmark Results: Synthetic Dataset (10,000 Vectors)

**Date**: 2026-01-18
**Environment**: Local Development (macOS/Arm64)
**Mode**: Release
**Version**: P10 Optimization Suite Enabled

## Configuration
- **Dataset**: Synthetic (Random Float32)
- **Dimension**: 128
- **Base Vectors**: 10,000
- **Queries**: 100
- **TopK**: 10
- **Concurrency**: 4

- **Rate**: 15,592.5 vec/s
- **Total Time**: 0.64s

## Analysis

### Comparison with 1k Vectors
| Metric | 1k Vectors | 10k Vectors | Change |
| :--- | :--- | :--- | :--- |
| QPS | 1,894 | 272.5 | -85.6% |
| P99 Latency | 3.05 ms | 21.43 ms | +7.0x |

### Key Observations
1.  **Expected QPS Degradation**: As dataset size increases 10x (1k â†’ 10k), QPS dropped significantly. This is expected with BruteForce/IVF-Flat algorithms where search time scales linearly with data size.
2.  **P99 Still Reasonable**: P99 latency of 21ms is within typical SLA requirements for many RAG applications, but approaching the edge of acceptable performance.
3.  **Load Throughput Improved**: Interestingly, the load throughput *increased* with more data. This is likely due to better amortization of connection overhead and batch processing efficiency.
4.  **Next Steps**: 
    - Consider enabling IVF indexing with `nlist` tuning for larger datasets.
    - Test with 100k vectors to identify the scalability ceiling.
    - Evaluate Scalar Quantization (SQ8) impact at this scale.
